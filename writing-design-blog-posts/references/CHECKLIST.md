# Pre-submission checklist

Run through every item before delivering the final draft. Fix any failures before presenting the article.

## Title and subtitle

- [ ] Title is **sentence case** (only first word + proper nouns capitalized)
- [ ] Title is 8–12 words, under 43 characters ideally
- [ ] Title signals a **specific point of view**, not just a topic
- [ ] Subtitle exists and is one clear sentence
- [ ] Subtitle is also sentence case
- [ ] No period at the end of the title

## Opening

- [ ] First 1–2 paragraphs contain a **personal hook** from real experience
- [ ] Hook names at least one of: a year, a project, a company, a specific situation
- [ ] No definition or dictionary-style opening
- [ ] No "In this article, we will explore..."
- [ ] Thesis or promise is clear within the first 3 paragraphs
- [ ] Main keyword appears in the first 100 words (for SEO)

## Structure

- [ ] Article has 5–7 H2 sections (fewer for career guides, more for long case studies)
- [ ] All headings are **descriptive and scannable** (not clever or cryptic)
- [ ] All headings use sentence case
- [ ] No bold, italic, or all-caps formatting in headings
- [ ] Subheadings appear every 300–500 words
- [ ] The article is **standalone** — no "Part 1 / Part 2" references

## Voice

- [ ] Opens with personal anecdote, not theory
- [ ] Explains "why" before "how" in every recommendation
- [ ] Uses first person ("I" / "we") naturally
- [ ] Contains at least one parenthetical aside for intimacy
- [ ] Contains at least one metaphor or vivid phrase
- [ ] Contains at least one honest admission (limitation, failure, surprise)
- [ ] **Passes the ChatGPT test**: would a reader get more from this than from asking an AI the same question?

## Anti-AI-slop check

- [ ] No "In today's fast-paced world..." or "In the world of..."
- [ ] No "Let's dive in" / "Without further ado"
- [ ] No "In conclusion" / "To sum up"
- [ ] No "It's important to note that..."
- [ ] No "game-changer" / "revolutionize" / "landscape"
- [ ] No encyclopedic paragraphs without personal stake
- [ ] No perfectly balanced "on the other hand" structures
- [ ] No "leverage" / "utilize" / "streamline" — use plain English

## Formatting

- [ ] Paragraphs are 2–4 sentences max
- [ ] Bullet lists use **bold lead-in phrases**
- [ ] Images have `![alt text](placeholder)` with descriptive alt text
- [ ] Case study images use `*Fig.N — Description*` caption format
- [ ] External links point to tools, resources, or evidence (not promotional)
- [ ] Max **2 exclamation marks** in the entire article
- [ ] Emojis only in career/guidelines content, never in case studies or opinion pieces

## Length and tags

- [ ] Word count is in the target range for the article type:
  - Standard article: 1,600–2,100 words
  - Case study: 2,500–3,500 words
  - Career guide: 1,000–1,800 words
- [ ] 5 topic tags recommended at the end
  - 2–3 broad tags (Design, UX, Product Design)
  - 2–3 specific tags matching the article's focus

## Closing

- [ ] Closing is **warm and encouraging**, not just a summary
- [ ] Uses "I hope" or similar generous language
- [ ] Includes a bonus resource, recommendation, or link
- [ ] Invites connection or action
- [ ] Acknowledges collaborators by name (if applicable)

## UX Collective compliance

- [ ] No promotional links, banners, or self-promotion in the body
- [ ] No calls for claps or comments
- [ ] Written for a **global audience** (no unexplained cultural references)
- [ ] Cover image specified or noted (1500×1000 px recommended)
- [ ] Article could not have been easily written by AI — unique voice is evident

## Final step

- [ ] Output is a complete `.md` file
- [ ] Tags included as an HTML comment at the end: `<!-- Tags: ... -->`
- [ ] Note included about Markdown-to-Medium import via GitHub Gist method
